The Filesystem API provides comprehensive file and directory management capabilities within the sandbox, including CRUD operations, permission management, search, and batch operations.

## Get File Metadata

<RequestExample>
```bash cURL Single File
curl -X GET "http://localhost:44772/files/info?path=/workspace/file.txt" \
  -H "X-EXECD-ACCESS-TOKEN: your-token"
```

```bash cURL Multiple Files
curl -X GET "http://localhost:44772/files/info?path=/workspace/file1.txt&path=/workspace/file2.py" \
  -H "X-EXECD-ACCESS-TOKEN: your-token"
```

```python Python
import requests

response = requests.get(
    "http://localhost:44772/files/info",
    headers={"X-EXECD-ACCESS-TOKEN": "your-token"},
    params={"path": ["/workspace/file.txt", "/workspace/data.csv"]}
)

files_info = response.json()
for path, info in files_info.items():
    print(f"{path}: {info['size']} bytes, mode {info['mode']}")
```
</RequestExample>

Retrieves detailed metadata for one or multiple files including permissions, owner, group, size, and modification time.

<ParamField query="path" type="array" required>
  File path(s) to get info for (can be specified multiple times)
</ParamField>

<ResponseField name="[path]" type="object">
  Map of file paths to FileInfo objects
  
  <Expandable title="FileInfo properties">
    <ResponseField name="path" type="string">
      Absolute file path
    </ResponseField>
    
    <ResponseField name="size" type="integer">
      File size in bytes
    </ResponseField>
    
    <ResponseField name="modified_at" type="string">
      Last modification time (RFC3339 format)
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      File creation time (RFC3339 format)
    </ResponseField>
    
    <ResponseField name="owner" type="string">
      File owner username
    </ResponseField>
    
    <ResponseField name="group" type="string">
      File group name
    </ResponseField>
    
    <ResponseField name="mode" type="integer">
      File permissions in octal format (e.g., 755)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json 200
{
  "/workspace/file.txt": {
    "path": "/workspace/file.txt",
    "size": 2048,
    "modified_at": "2025-11-16T14:30:45Z",
```

## Search for Files

<RequestExample>
```bash cURL All Files
curl -X GET "http://localhost:44772/files/search?path=/workspace" \
  -H "X-EXECD-ACCESS-TOKEN: your-token"
```

```bash cURL Pattern
curl -X GET "http://localhost:44772/files/search?path=/workspace&pattern=**/*.py" \
  -H "X-EXECD-ACCESS-TOKEN: your-token"
```

```python Python
import requests

response = requests.get(
    "http://localhost:44772/files/search",
    headers={"X-EXECD-ACCESS-TOKEN": "your-token"},
    params={
        "path": "/workspace",
        "pattern": "**/*.py"
    }
)

files = response.json()
for file in files:
    print(f"{file['path']} ({file['size']} bytes)")
```
</RequestExample>

Searches for files matching a glob pattern within a specified directory and its subdirectories. Returns file metadata including path, permissions, owner, and group.

<ParamField query="path" type="string" required>
  Root directory path to search in
</ParamField>

<ParamField query="pattern" type="string" default="**">
  Glob pattern to match files. Supports patterns like:
  - `**` - All files recursively
  - `*.txt` - All .txt files in directory
  - `**/*.py` - All .py files recursively
  - `test_*.py` - Files starting with "test_"
</ParamField>

<ResponseField name="files" type="array">
  Array of matching FileInfo objects
  
  <Expandable title="FileInfo properties">
    <ResponseField name="path" type="string">

    </ResponseField>

    <ResponseField name="group" type="string">
      File group name
    </ResponseField>
    
    <ResponseField name="mode" type="integer">
      File permissions in octal format

</ResponseField>

## Upload Files

<RequestExample>
```bash cURL
curl -X POST http://localhost:44772/files/upload \
  -H "X-EXECD-ACCESS-TOKEN: your-token" \
  -F 'metadata={"path":"/workspace/file.txt","owner":"admin","group":"admin","mode":644}' \
  -F 'file=@/local/path/file.txt'
```

```python Python
import requests
import json

metadata = {
    "path": "/workspace/upload.txt",
    "owner": "admin",
    "group": "admin",
    "mode": 644
}

files = {
    'metadata': (None, json.dumps(metadata), 'application/json'),
    'file': ('upload.txt', open('/local/path/file.txt', 'rb'), 'application/octet-stream')
}

response = requests.post(
    "http://localhost:44772/files/upload",
    headers={"X-EXECD-ACCESS-TOKEN": "your-token"},
    files=files
)

if response.status_code == 200:
    print("File uploaded successfully")
```
</RequestExample>

Uploads one or multiple files to specified paths within the sandbox. Uses multipart form data with metadata and file content.

<ParamField body="metadata" type="string" required>
  JSON-encoded file metadata
  
  <Expandable title="Metadata properties">
    <ParamField body="path" type="string" required>
      Target file path in the sandbox
    </ParamField>
    
    <ParamField body="owner" type="string">
      File owner username
    </ParamField>
    
    <ParamField body="group" type="string">
      File group name
    </ParamField>
    
    <ParamField body="mode" type="integer">
      File permissions in octal format
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="file" type="binary" required>
  File content (binary)
</ParamField>

## Download File

<RequestExample>
```bash cURL
curl -X GET "http://localhost:44772/files/download?path=/workspace/data.csv" \
  -H "X-EXECD-ACCESS-TOKEN: your-token" \
  -o data.csv
```

```bash cURL Partial (Range)
curl -X GET "http://localhost:44772/files/download?path=/workspace/large.bin" \
  -H "X-EXECD-ACCESS-TOKEN: your-token" \
  -H "Range: bytes=0-1023" \
  -o partial.bin
```

```python Python
import requests

response = requests.get(
    "http://localhost:44772/files/download",
    headers={"X-EXECD-ACCESS-TOKEN": "your-token"},
    params={"path": "/workspace/data.csv"},
    stream=True
)

if response.status_code == 200:
    with open('data.csv', 'wb') as f:
        for chunk in response.iter_content(chunk_size=8192):
            f.write(chunk)
    print("File downloaded successfully")
```
</RequestExample>

Downloads a file from the specified path within the sandbox. Supports HTTP range requests for resumable downloads and partial content retrieval.

<ParamField query="path" type="string" required>
  Absolute or relative path of the file to download
</ParamField>

<ParamField header="Range" type="string">
  HTTP Range header for partial content requests (e.g., "bytes=0-1023")
</ParamField>

<ResponseField name="body" type="binary">
  File content as binary stream
</ResponseField>

<ResponseField name="Content-Disposition" type="string" header>
  Attachment header with filename
</ResponseField>

<ResponseField name="Content-Length" type="integer" header>
  File size in bytes (or range size for partial requests)
</ResponseField>

<ResponseField name="Content-Range" type="string" header>
  Range of bytes being returned (only for 206 responses)
</ResponseField>

<ResponseExample>
```text 200
[Binary file content]
```

```text 206 Partial Content
[Partial binary file content]
```

```json 404
{
  "code": "FILE_NOT_FOUND",
  "message": "file not found"

```

```json 416
{
  "code": "RANGE_NOT_SATISFIABLE",

}
```
</ResponseExample>

## File Permissions

Unix-style file permissions use octal notation:

| Octal | Binary | Permissions | Description |
|-------|--------|-------------|-------------|
| 755 | 111 101 101 | rwxr-xr-x | Owner: read+write+execute, Group/Others: read+execute |
| 644 | 110 100 100 | rw-r--r-- | Owner: read+write, Group/Others: read only |
| 600 | 110 000 000 | rw------- | Owner: read+write, Group/Others: no access |
| 777 | 111 111 111 | rwxrwxrwx | Everyone: full access (use with caution) |

### Permission Breakdown

- **First digit**: Owner permissions
- **Second digit**: Group permissions
- **Third digit**: Others permissions

Each digit is the sum of:
- **4** = Read (r)
- **2** = Write (w)
- **1** = Execute (x)

### Examples

```json
// Executable script
{
  "mode": 755,  // Owner can modify, everyone can read/execute
  "owner": "admin",
  "group": "admin"
}

// Configuration file
{
  "mode": 644,  // Owner can modify, everyone can read
  "owner": "admin",
  "group": "admin"
}

// Private key
{
  "mode": 600,  // Only owner can read/write
  "owner": "admin",
  "group": "admin"
}
```
